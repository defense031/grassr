# grass v0.2.0 — paper alignment design doc

**Status.** Draft for redline. No R code touched yet.
**Target.** Bring the package to the state the paper's §3 Quick Start describes, so a reader who installs `defense031/grass@v0.2.0` and runs the paper's Quick Start verbatim succeeds.
**Date.** 2026-05-02.
**Sequencing.** This doc is S1 of a 6-session package sprint (S1 audit + spec → S2-S5 implementation → S6 cleanup + release).

---

## 0. Design directives (from user, 2026-05-02)

1. **Rating matrix in, Report Card out.** Three lines of R, one matrix, one printed object. The user does not compute pi_hat, k, N, the panel of coefficients, the cross-coefficient spread, or the per-rater Se/Sp before calling the package.
2. **Visceral.** A reader who has never touched R should look at the Quick Start and immediately understand the call. Function names read as English. Argument names read as English. Default behavior is the right behavior 90% of the time.
3. **Layered access.** The Report Card displays a curated four-field summary, but `summary()`, `as.data.frame()`, and `plot()` give the user the full panel of coefficients, all percentiles, bootstrap CIs, nearest-neighbor caveats, band probabilities, and surface-position visualization on demand. Nothing computed and discarded.
4. **Self-documenting.** A Python user reading `grass_report(ratings = Y)` should understand the call without consulting docs. R idiom is fine; R-specific obscurity is not.

These directives govern every choice below. When the spec proposes alternatives, the criterion is which best satisfies (1)-(4).

## 2. Headline API (the paper's Quick Start)

```r
library(grass)
Y      <- <N x k binary rating matrix>
report <- grass_report(ratings = Y)
report                         # prints the four-field Report Card
summary(report)                # full panel, percentiles, CIs, caveats
as.data.frame(report)          # tidy long-format for further analysis
plot(report)                   # surface visualization with the panel pinned
```

**Headline call: one function, one argument.** Auto-derives pi_hat from rowSums, k from ncol, N from nrow. Auto-picks the primary coefficient by (k, pi_hat) per Table 2 of the paper. Auto-routes to per-rater output if δ̂ ≥ 11.75. The user sees the Report Card; everything underneath is on demand.

**Granular calls (same workflow, more control).** All exported. Each takes `ratings = Y` as the primary path; each has scalar-input audit signatures preserved.
```r
position_on_surface(ratings = Y, metric = "pabak")   # one coefficient, one position
check_asymmetry(ratings = Y)                          # cross-coefficient δ̂ + flag
latent_class_fit(ratings = Y)                         # per-rater Ŝe, Ŝp + bootstrap CI
compute_panel(ratings = Y)                            # the panel of observed coefficients
```

**Power-user knobs (named arguments, sensible defaults).**
- `metric = "auto"` (default) | `"pabak"` | `"ac1"` | `"fleiss_kappa"` | `"krippendorff_a"` | `"icc"` — override the auto-pick
- `axis = "inter"` (default) | `"intra"` — switches the DGP / surface family
- `bands = c(0.5, 0.625, 0.75, 0.875, 1.0)` — partition for the four-band label
- `band_labels = c("Poor", "Moderate", "Strong", "Excellent")` — partition labels
- `delta_thresholds = c(9.25, 11.75)` — the (caution, divergent) cutoffs in pp
- `bootstrap_B = 1000L` — bootstrap replicates for divergent-branch CIs
- `verbose = FALSE` — emit progress messages on long calls

## 3. Class & S3 method design (layered access)

### 3.1 The headline class: `grass_card`

`grass_report()` returns an object of class `c("grass_card", "list")`. Internal structure:

```r
structure(
  list(
    # ---- The four printed fields ----
    sample = list(
      k = <integer>,                   # n raters
      N = <integer>,                   # n subjects
      pi_hat = <numeric in (0,1)>,     # observed positive rate
      tau2_hat = <numeric or NA>,      # subject-variance estimate (ICC; NA otherwise)
      axis = "inter" | "intra"
    ),
    coefficient = list(
      primary = <character>,            # "pabak", "ac1", etc.
      observed_value = <numeric>,
      surface_percentile = <numeric in [0,100]>,
      band = <character>,               # "Poor" / "Moderate" / "Strong" / "Excellent"
      qualifier = <character>           # "decisive" / "moderate" / "weak"
    ),
    delta = list(
      delta_hat = <numeric>,            # in pp
      flag = <character>,               # "aligned" / "caution" / "divergent"
      thresholds = c(caution = 9.25, divergent = 11.75)
    ),
    # ---- The full underlying panel (always populated) ----
    panel = data.frame(
      coefficient        = c("pabak", "ac1", "fleiss_kappa", "krippendorff_a", "icc"),
      observed_value     = ...,
      surface_percentile = ...,
      band               = ...,
      qualifier          = ...,
      band_probability_modal = ...,    # the modal-band sampling probability
      ci_lower           = ...,         # Wilson-logit CI on observed value
      ci_upper           = ...,
      nn_distance_k      = ...,         # nearest-neighbor distance in k-grid (0 if exact)
      nn_distance_N      = ...,         # nearest-neighbor distance in N-grid
      notes              = list(...)    # per-coefficient caveat strings
    ),
    # ---- Divergent-branch payload (NULL when flag != "divergent") ----
    per_rater = NULL | data.frame(      # populated only when flag == "divergent"
      rater   = character,               # "R1", "R2", ...
      se_hat  = numeric,
      sp_hat  = numeric,
      se_lower = numeric,                # bootstrap CI lower
      se_upper = numeric,
      sp_lower = numeric,
      sp_upper = numeric,
      bound_only = logical               # TRUE when k=2 (range, not point)
    ),
    # ---- Reference-surface artifacts (for plot() and summary()) ----
    surface = list(
      q_grid             = numeric vector (length 501),
      reference_curves   = list(<one entry per panel coefficient>: numeric length 501),
      reference_type     = "fitted" | "oracle" | "closed-form",  # per coefficient
      band_boundaries_q  = c(0.5, 0.625, 0.75, 0.875, 1.0)
    ),
    # ---- Metadata ----
    call = <match.call()>,
    grass_version = packageVersion("grass"),
    timestamp = Sys.time(),
    inputs = list(                      # echo the inputs for audit
      ratings_dim = c(N, k),
      ratings_md5 = digest::digest(ratings),
      axis = ...,
      delta_thresholds = ...,
      bands = ...,
      band_labels = ...
    )
  ),
  class = c("grass_card", "list")
)
```

**Why all of this lives in one object.** The user's directive: "the report card might USE a ton of information but not all of it gets immediately reported to the user. But if they want to see it, it should be available." The headline `print()` shows `sample`, `coefficient`, `delta` (the four-field card). `summary()` shows `panel` + `per_rater` + caveats. `as.data.frame()` returns `panel` plus the divergent-branch per-rater table. `plot()` uses `surface` + `panel`. Reproducibility checks read `inputs` + `grass_version`. Nothing is recomputed; everything is one allocation.

### 3.2 S3 methods on `grass_card`

| Method | What it returns / shows | Output |
|--------|--------------------------|--------|
| `print(x)` | The four-field Report Card, formatted to match the paper's Figure 2 right panel character-for-character | Console block ~10 lines |
| `format(x)` | Same content as `print()` but as a character vector for embedding | `character()` |
| `summary(x)` | Full panel table + per-rater CIs (if divergent) + caveats + nearest-neighbor distances + grass version | Console multi-section block |
| `print(summary(x))` | Pretty-print of the summary | Console block |
| `as.data.frame(x)` | The `panel` data frame in long format, one row per (coefficient × {observed, percentile, ci_lower, ci_upper, ...}) — tidy-ready | `data.frame` |
| `tidy(x)` | Same as `as.data.frame` but with `broom`-style column names | `data.frame` |
| `plot(x)` | Surface visualization (default panel: see §3.3) | `ggplot` object |
| `plot(x, type = ...)` | Alternative views (see §3.3) | `ggplot` object |

### 3.3 `plot.grass_card` views

Default view: **the surface for the primary coefficient, with the observation pinned, and the four-band partition shaded.** A 2D heatmap of E[primary metric] over (M_1, q) at the study's (k, N), with a single solid white point at the observed (pi_hat, q_hat) and dotted contour lines at q ∈ {0.5, 0.625, 0.75, 0.875, 1.0} demarcating the bands. The visceral artifact: the user sees their study's position on a calibrated reference surface in one image.

Alternative views via `type = ...`:

| `type` | Shows |
|--------|-------|
| `"surface"` (default) | Primary-coefficient surface with observation pinned |
| `"panel"` | All 4-5 coefficients on a single coordinate (percentile axis) so the cross-coefficient spread is visible |
| `"thermometer"` | Replicates the §3 thermometer figure with `delta_hat` on it; pure δ̂ visualization |
| `"intervals"` | Forest plot of observed coefficients with their 95% CIs |
| `"per_rater"` | When divergent: forest plot of per-rater Ŝe and Ŝp with bootstrap CIs |
| `"diagnostic"` | `patchwork` composite of "panel" + "thermometer" + "per_rater" (when applicable) — full diagnostic at-a-glance |

All `plot()` types return ggplot objects so the user can override theme / colors / titles / save.

## 4. Function-by-function spec

### 4.1 `grass_report(ratings, axis = "inter", ...)`

**Signature.**
```r
grass_report <- function(ratings,
                         axis            = c("inter", "intra"),
                         metric          = "auto",
                         occasion        = NULL,             # for axis = "intra"
                         bands           = c(0.5, 0.625, 0.75, 0.875, 1.0),
                         band_labels     = c("Poor", "Moderate", "Strong", "Excellent"),
                         delta_thresholds = c(9.25, 11.75),
                         bootstrap_B     = 1000L,
                         verbose         = FALSE,
                         ...) { ... }
```

**Body sketch.**
1. **Validate and normalize input.** Accept N×k integer matrix, data.frame with N rows × k rater columns, or paired list. Coerce to canonical N×k integer matrix `Y`. Reject if values aren't in {0, 1}, if k < 2, if N < 10 (warn at N < 30).
2. **Auto-derive context.** `pi_hat <- mean(Y)`, `k <- ncol(Y)`, `N <- nrow(Y)`. For `axis = "intra"`: require `occasion` (vector or factor identifying viewing); reshape into the (subject × rater × viewing) cube; `tau2_hat` from the GLMM fit.
3. **Compute the panel.** `panel_obs <- compute_panel(Y, axis)` → named list of the 4-5 observed coefficients. (At k = 2: only PABAK, AC1, κ; at k ≥ 3: add Fleiss κ, Krippendorff α, ICC.)
4. **Per-coefficient surface positioning.** For each coefficient `m` in the panel, call `position_on_surface(ratings = Y, metric = m, ...)` → list with `surface_percentile`, `band`, `qualifier`, `band_probability_modal`, `q_hat`, `notes`. Also pull a Wilson-logit CI on the observed value.
5. **Pick primary coefficient.** `primary <- if (metric == "auto") pick_primary_coefficient(k, pi_hat, axis) else metric`. Validate that `primary ∈ panel`.
6. **Cross-coefficient δ̂.** `delta_hat <- (max - min) of surface_percentile across panel, in pp`. `flag <- if delta_hat < 9.25 "aligned" else if < 11.75 "caution" else "divergent"`.
7. **Divergent branch.** If `flag == "divergent"`:
   - If `k >= 3`: `per_rater <- latent_class_fit(Y, B = bootstrap_B)` → data.frame.
   - If `k == 2`: `per_rater <- latent_class_fit(Y, B = bootstrap_B)` returns bounds with `bound_only = TRUE`.
   - The `coefficient$band` and `coefficient$qualifier` are set to `NA` (the §3 "band suppressed" rule); the printed coefficient line still shows the primary's percentile but the band field reads `suppressed`.
8. **Assemble the `grass_card` object.** Populate every field per §3.1. Attach `surface$reference_curves` from `position_on_surface` calls.
9. **Return.**

**Error modes.**
- `ratings` not a matrix → coerce or error with guidance.
- All-zero or all-one column → warn (no signal in this rater).
- N < 10 → warn that inference is unreliable.
- (k, N, pi_hat) outside the calibrated envelope → note in `panel$notes` and the printed output (nearest-neighbor distance ≥ some threshold).

### 4.2 `position_on_surface(ratings, metric, ...)` — additive rewrite

Add a `ratings` primary path that auto-derives `obs_value`, `pi_hat`, `k`, `N`. Existing scalar path preserved.

```r

                                N         = NULL,            # was required; now NULL-allowed if `ratings` given
                                method    = c("empirical", "delta"),
                                bands     = c(0.5, 0.625, 0.75, 0.875, 1.0),

                                ratings   = NULL,
                                reference_type = c("fitted", "oracle"),

  # NEW: ratings-primary branch
  if (!is.null(ratings)) {

    Y <- normalize_ratings(ratings)
```

### 4.3 `check_asymmetry(ratings, ...)` — new function (rename old)

**Rename current `check_asymmetry()` to `check_rater_asymmetry()`.** Same signature, same behavior. Still useful in the divergent branch where per-rater Se/Sp are available. Emit a one-time soft deprecation if user calls `check_asymmetry(se = ..., sp = ...)` (the old form), pointing them to either `check_rater_asymmetry()` (if they want the per-rater gap) or `check_asymmetry(ratings = Y)` (the new convention).

**New `check_asymmetry(ratings, ...)`.**
```r
check_asymmetry <- function(ratings,
                            axis = c("inter", "intra"),
                            occasion = NULL,
                            delta_thresholds = c(9.25, 11.75),
                            ...) {
  Y <- normalize_ratings(ratings)
  axis <- match.arg(axis)
  panel_obs <- compute_panel(Y, axis = axis, occasion = occasion)
  positions <- lapply(names(panel_obs), function(m) {
    position_on_surface(ratings = Y, metric = m)
  })
  pcts <- vapply(positions, `[[`, numeric(1), "percentile") * 100  # to pp
  delta_hat <- max(pcts) - min(pcts)
  flag <- if (delta_hat < delta_thresholds[1]) "aligned"
          else if (delta_hat < delta_thresholds[2]) "caution"
          else "divergent"
  out <- list(
    delta_hat = delta_hat,
    flag = flag,
    thresholds = setNames(delta_thresholds, c("caution", "divergent")),
    panel = data.frame(
      coefficient = names(panel_obs),
      observed = unlist(panel_obs),
      percentile_pp = pcts
    ),
    notes = unique(unlist(lapply(positions, `[[`, "notes")))
  )
  class(out) <- c("grass_asymmetry_panel", "list")
  out
}
```

Returns a `grass_asymmetry_panel` object (new class — distinct from old `grass_asymmetry` so we can keep the rename clean). `print.grass_asymmetry_panel` shows δ̂, flag, threshold, and the panel breakdown.

### 4.4 `latent_class_fit(ratings, B = 1000L, ...)` — new

Implementation: Dawid-Skene EM with binary class C (k columns, N subjects). Initialize Ŝe and Ŝp at majority-vote consensus; iterate E (P(C_i = 1 | Y_i)) and M (Ŝe_j, Ŝp_j) until convergence. Wraps in nonparametric bootstrap CI (resample subjects with replacement B times; refit; report 2.5/97.5 percentiles). At k = 2: closed-form bounds rather than point estimate (cite Hui-Walter 1980 in the Roxygen).

```r
latent_class_fit <- function(ratings,
                             B = 1000L,
                             method = c("dawid_skene_em", "hui_walter"),  # k=2 dispatch
                             max_iter = 1000L,
                             tol = 1e-6,
                             seed = NULL,
                             ...) {
  Y <- normalize_ratings(ratings)
  k <- ncol(Y)
  if (k < 2) stop("Need at least 2 raters.")
  if (k == 2) return(latent_class_k2_bounds(Y, B = B, seed = seed))
  # k >= 3: Dawid-Skene EM
  fit <- ds_em(Y, max_iter = max_iter, tol = tol)
  if (B > 0) {
    boot <- ds_em_bootstrap(Y, B = B, seed = seed)
    fit$ci <- boot$ci
  }
  class(fit) <- c("grass_latent_class", "list")
  fit
}
```

Optional dependency: `lme4` is already in Suggests for the GLMM ICC path; the EM is hand-rolled (~50 lines), no new dep.

### 4.5 `compute_panel(ratings, axis = "inter", occasion = NULL)` — new internal

Helper that returns the full panel of observed coefficients from an N×k matrix:
```r
compute_panel <- function(ratings, axis = "inter", occasion = NULL) {
  Y <- normalize_ratings(ratings)
  k <- ncol(Y)
  out <- list()
  if (k == 2) {
    # Two-rater path: fall through to existing metrics-core, plus alpha
    tab <- build_table(Y[, 1], Y[, 2])
    base <- compute_agreement_metrics(tab)
    out$pabak <- base[["PABAK"]]
    out$ac1   <- base[["AC1"]]
    out$kappa <- base[["kappa"]]
    out$krippendorff_a <- obs_krippendorff_alpha(Y)
    # ICC at k=2 is computable but uncommon for binary; defer
  } else {
    out$pabak <- obs_mean_pairwise_pabak(Y)
    out$ac1   <- obs_mean_pairwise_ac1(Y)
    out$fleiss_kappa <- obs_fleiss_kappa(Y)
    out$krippendorff_a <- obs_krippendorff_alpha(Y)
    out$icc <- obs_icc_glmer(Y)   # ICC(1,1) GLMM fit; returns NA if lme4 unavailable
  }
  out
}
```

The `obs_*` helpers are ports of `paper2/code/02_metrics_observed.R` into `R/metrics-multi-rater.R` (new file). Internal only (not exported).

### 4.6 `pick_primary_coefficient(k, pi_hat, axis)` — new internal

Implements Table 2 of the paper:
```r
pick_primary_coefficient <- function(k, pi_hat, axis = "inter") {
  if (axis == "intra") return("icc")    # ICC(3,1) per Koo & Li
  if (k == 2) {
    # PABAK at balanced; AC1 at extreme prevalence
    return(if (pi_hat < 0.2 || pi_hat > 0.8) "ac1" else "pabak")
  }
  # k >= 3: PABAK at balanced; AC1 at extreme prevalence (per App F evidence)
  if (pi_hat < 0.15 || pi_hat > 0.85) "ac1" else "pabak"
}
```

Hard-coded thresholds chosen to align with App F's observed crossover. Configurable via `...` if the user wants to override (advanced).

```
  See `summary(...)` for full panel and CI details.
  See `plot(...)` for a surface-position visualization.
```

`summary.grass_card`. Returns a `summary.grass_card` list with the full panel; `print.summary.grass_card` formats to a multi-section block (panel table, divergent-branch per-rater table when applicable, caveats list).

`as.data.frame.grass_card`. Returns `x$panel` augmented with the four-field summary as the first rows in a single tidy data frame.

`plot.grass_card`. See §3.3.

## 6. Test plan

### 6.1 Unit tests

For each new / rewritten function, at least:
- Call with valid input → returns expected class.
- Call with degenerate input (all-zero column, k=2, N=10, pi_hat at 0.05 / 0.95) → expected behavior.
- Call with invalid input → informative error.

### 6.2 Round-trip tests

- `grass_report(ratings = Y)` → primary coefficient's percentile equals `position_on_surface(ratings = Y, metric = primary)$percentile`.
- `compute_panel(Y)$pabak` equals `obs_mean_pairwise_pabak(Y)`.
- Scalar-path `position_on_surface(obs_value = ..., pi_hat = ..., k = ..., N = ...)` equals ratings-path `position_on_surface(ratings = Y)` to floating point.

### 6.3 Paper-claim integration tests

These are the load-bearing ones. The paper makes specific quantitative claims; the package must reproduce them.

- **§4 worked example.** Build a synthetic 5×200 binary matrix with the shared-bias structure described in §4 (Se=0.93, Sp=0.67, all five raters; pi_hat=0.12). Call `grass_report(ratings = Y)`. Verify:
  - `panel$surface_percentile` for `kappa` is in [45, 51] (paper says 48; allow MC slack)
  - same for PABAK (paper 30, allow [27, 33])
  - same for AC1 (paper 70, allow [67, 73])
  - same for α (paper 48, allow [45, 51])
  - `delta_hat` is in [37, 43] (paper 40)
  - `flag == "divergent"`
  - `per_rater$se_hat` for each rater is in [0.88, 0.97] (paper says cluster ≈ 0.93)
  - `per_rater$sp_hat` for each rater is in [0.60, 0.74] (paper says cluster ≈ 0.67)

- **§4 counter-example (k=2 symmetric).** Build the 2×200 case from `tab:k2_sym_crosstab` (cells 180/5/7/8). Call `grass_report(ratings = Y)`. Verify:
  - PABAK observed equals 0.88 ± 0.01
  - PABAK percentile in [89, 95] (paper 92)
  - flag == "aligned"
  - delta_hat in [1, 5] pp (paper 3)
  - band == "Excellent", qualifier == "decisive"

- **App G operating characteristics.** Replicate one row of `tab:delta_oc`: at k=5, N=200, A=0.20, with the specified DGP, the rejection rate at the 12-pp threshold should be 0.30 ± 0.03 (paper says 0.301 ± MC).

### 6.4 sysdata.rda coverage audit

A test that loads sysdata and asserts:
- `empirical_q_hat_surface` covers k ∈ {2, 3, 5, 8, 15, 25}, N ∈ {50, 200, 1000}, 52 F-keys per (k, N), all five metrics.
- `fitted_icc_reference_curves` covers k ∈ {3, 5, 8, 15, 25}, N ∈ {50, 200} (per the paper's "fitted unavailable at N=1000" caveat).
- `icc_reference_curves` covers the oracle slice for the same grid.

Anything missing → fail the test, file an issue, document in NEWS.md.

### 6.5 CRAN-readiness checks

(For S6.) `R CMD check --as-cran` clean. `goodpractice::gp()` reasonable. No mistakenly-exported internals. No cross-package dependencies that aren't in DESCRIPTION.

## 9. Open questions for user redline

These are the design choices I made above that are reversible and worth explicit user sign-off before implementation begins.

**Q1. Class name: `grass_card` or keep `grass_result`?**
Spec proposes `grass_card` (new class) so the old `grass_result` regime/distance/PI-BI machinery can be deprecated cleanly without a class collision. Alternative: rename `grass_result` → `grass_card` and migrate its slots. I lean `grass_card`-new for clean separation; you might prefer to reuse the class name for backward compat.

**Q2. Function name for the new percentile-spread δ̂: `check_asymmetry` (per the paper) or something less collide-prone like `check_panel_disagreement`?**
The paper currently writes `check_asymmetry(ratings = Y)` in §3 Quick Start (`03_framework.tex:124`). The package's existing `check_asymmetry(se, sp, ...)` is the OLD framework. Spec proposes: rename old to `check_rater_asymmetry`, take over `check_asymmetry` for the new function. Risk: an existing user who installed grass 0.1.x and is upgrading will get a soft-deprecation hint, not silent breakage. Alternative: pick a new name (`check_panel_disagreement`?) and update the paper's Quick Start instead.

**Q3. Headline function: `grass_report()` (rewrite body) or new name (`grass_card()`)?**
Spec proposes rewriting `grass_report()`'s body. Risk: any user who installed grass 0.1.x and called `grass_report(2x2_table, format = "matrix")` will get "argument 'data' is missing" — failure with a clear error, not silent wrong answer. Alternative: keep the old `grass_report()` callable for one cycle (with a deprecation hint) and put the new headline at `grass_card()`. Cleaner for migration; uglier for the paper's Quick Start.

**Q4. Default metric: `metric = "auto"` or require explicit?**
Spec proposes `metric = "auto"` (default), which calls `pick_primary_coefficient()`. Pro: visceral — user supplies one matrix, gets a single answer. Con: hides a design choice. Alternative: require `metric = "pabak"` etc. explicitly. I lean `auto` per design directive (1).

**Q5. `axis = "intra"` and `occasion = ...` in v0.2.0 or v0.3.0?**
The paper covers both inter and intra. Inter is the headline; intra is the secondary axis with §4's last summary-table row + Appendix B. Spec proposes including intra in v0.2.0 (the paper claims it). Risk: ICC(3,1) variant selection on the intra axis is its own complexity; could add S5.5 to the sprint. Alternative: ship inter-only at v0.2.0; intra at v0.2.1 with a clear "intra axis: planned for v0.2.1" note in v0.2.0's docs.

**Q6. Latent-class implementation: hand-rolled EM or external dep (`randomLCA`)?**
Spec proposes hand-rolled Dawid-Skene EM (~50 lines, no new dep). Pro: total control; no version-skew with an external package. Con: have to maintain. Alternative: depend on `randomLCA` or `BayesLCA`. Hand-rolled is my preference for a methods-paper companion package.

**Q7. Backward compat for old `grass_report(2x2_table, format = "matrix")` calls?**
Spec proposes hard break (the function does something new). Old users who upgrade will see a clear error from the new arg validator. Alternative: detect 2x2 input shape and route to a deprecation path that still computes the OLD framework output one more time with a deprecation warning. Adds ~30 lines of legacy code to maintain. I lean hard break: the package is pre-CRAN, paper is unsubmitted, no installed-base claim.

**Q8. plot() default view: surface-with-pin (visceral) or panel-on-percentile-axis (informative)?**
Spec proposes surface-with-pin as default. The §3 thesis is "position on surface"; the visualization should make that visceral. Alternative: default to the panel view which shows the cross-coefficient spread story.
