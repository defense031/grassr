# v0.5.3 design note — q_true interpolation in lookup_empirical_q_hat()

**Status.** Design note for review. No code changes yet. Targets
the next minor release after v0.5.2 ships the k = 2 ordering bug
fix + q-grid augmentations. Solves a structural class of issue
that ever-denser q-grids cannot fully eliminate.

## Problem

`grass/R/position_on_surface.R::lookup_empirical_q_hat()` (L683-760)
currently snaps q_hat to the nearest q_true in the bundled
sysdata grid:

```r
q_grid_sim <- sort(unique(sub2$q_true))
q_true_near <- q_grid_sim[which.min(abs(q_grid_sim - as.numeric(q_hat)))]
row_idx <- which(sub2$q_true == q_true_near)[1L]
```

The percentile is then computed against the empirical q-hat
distribution at *one* q_true cohort. When q_hat falls between two
adjacent grid points and the per-cohort distributions are tighter
than the grid spacing, q_hat lands in a "gap" between cohorts.

**Empirical pathology** (k = 25, N = 1000, pi_hat = 0.30,
PABAK = 0.78 → q_hat = 0.9416, on the v0.5.1 sysdata before the
k = 25 augmentation):

| q_true cohort | empirical q-hat distribution | q_hat = 0.9416 → percentile |
|---|---|---|
| 0.90 | q05 = 0.8969, q95 = 0.9029 | upper tail (~99) |
| 0.95 | q05 = 0.9477, q95 = 0.9523 | lower tail (~0) |

q_hat = 0.9416 is in the gap between 0.9029 (top of q_true = 0.90
cohort) and 0.9477 (bottom of q_true = 0.95 cohort). The lookup
snaps to q_true_near = 0.95 (because 0.9416 is closer to 0.95
than to 0.90 in q-distance) and returns percentile = 0 — i.e.
"this panel is in the bottom 1% of q = 0.95 panels," which is
literally true but substantively wrong because the panel is not
from the q = 0.95 cohort at all.

The same pathology hits at k = 2 N = 200 (fixed in v0.5.2 by
adding q = 0.97 to the k = 2 grid) and at k = 25 (addressed in
v0.5.2 by adding q ∈ {0.92, 0.97} to the k = 25 grid). It will
recur at any (k, N) cell whose per-cohort q-hat sd is smaller
than the q-grid spacing of 0.05.

## Why ever-denser q-grids cannot fully fix it

At k = 25, N = 1000, the closed-form-q̂ sd is roughly 0.003 (panels
are tightly informative). To have adjacent cohorts overlap at q05
and q95 with grid spacing of 0.05, you need each cohort's
two-sided 95% range to span at least 0.025 — so sd ≥ 0.006.
v0.5.2's k = 25 q = {0.92, 0.97} augmentation halves the grid
spacing to ~0.025-0.03 in the worst region; the boundary risk
remains at the corner cases (k = 25, N = 1000, q in the
~0.92-0.97 region with extreme F-keys).

**Densification is asymptotic patching.** It works for the cells
the paper actually uses, and doesn't generalize. A practitioner
running a k = 50 N = 5000 panel could rediscover the same
pathology at any q_hat between two grid points.

## Proposed v0.5.3 fix

**Bilinear interpolation in (q_true, percentile) space.** Replace
the "snap to nearest q_true" step with:

1. Find the two adjacent q_true cohorts that bracket q_hat (call
   them q_lo and q_hi).
2. Compute the percentile of q_hat against each cohort's empirical
   distribution: pct_lo, pct_hi.
3. Interpolate linearly:
   `pct = pct_lo + (q_hat - q_lo) / (q_hi - q_lo) * (pct_hi - pct_lo)`

Edge cases:
- q_hat below the lowest q_true → snap to lowest cohort, use
  pct directly (current behavior at the boundary).
- q_hat above the highest q_true → snap to highest cohort, use
  pct directly.
- q_hat exactly at a grid point → no interpolation (q_lo = q_hi).

**Why this works.** The percentile against the q_lo cohort
answers "where does this panel sit if its true quality is q_lo";
the percentile against q_hi answers the same for q_hi. If q_hat
is between, the panel's most defensible "where does it sit"
interpretation is the convex combination of the two cohort-conditional
readings, weighted by the panel's q-distance to each cohort. At
boundary q_hats the interpolation collapses to the current snap
behavior.

**Properties this preserves:**
- Monotone in q_hat (interpolation is linear in q_hat).
- Stable at grid points (interpolation collapses to the
  snapped value).
- Bounded in [0, 1] (each component pct is bounded).
- Doesn't change the q_hat itself (only the percentile readout).

**Properties this gains:**
- Smooth across q_true grid boundaries — no discrete jumps from
  ~0 to ~99 across infinitesimal q_hat changes.
- Robust to per-cohort tight distributions (the structural cause
  of the boundary-snap pathology).
- Independent of grid spacing — works at any q-grid density.

## Reference implementation sketch

```r
lookup_empirical_q_hat <- function(metric, pi_hat, k, N, q_hat) {
  # ... unchanged: surface fetch, k_near, n_near, F_key_near snap

  sub2 <- sub[sub$F_key == F_key_near, ]
  q_grid_sim <- sort(unique(sub2$q_true))
  q_hat_num <- as.numeric(q_hat)

  # NEW: bracket q_hat between two adjacent q_true cohorts
  if (q_hat_num <= min(q_grid_sim)) {
    q_lo <- q_hi <- min(q_grid_sim)
    w <- 0
  } else if (q_hat_num >= max(q_grid_sim)) {
    q_lo <- q_hi <- max(q_grid_sim)
    w <- 1
  } else {
    q_hi_idx <- which(q_grid_sim > q_hat_num)[1L]
    q_lo <- q_grid_sim[q_hi_idx - 1L]
    q_hi <- q_grid_sim[q_hi_idx]
    w <- (q_hat_num - q_lo) / (q_hi - q_lo)
  }

  fetch_pct <- function(q_target) {
    sid <- sub2$scenario_id[sub2$q_true == q_target][1L]
    row_in_arr <- which(as.integer(dimnames(surf$quantiles)$scenario_id)
                        == as.integer(sid))
    quants <- surf$quantiles[row_in_arr, metric, ]
    empirical_cdf_at(q_hat_num,
                     quantiles = quants,
                     probs = surf$probs)
  }

  pct_lo <- fetch_pct(q_lo)
  pct_hi <- fetch_pct(q_hi)
  percentile <- (1 - w) * pct_lo + w * pct_hi

  list(
    percentile = percentile,
    quantiles  = NULL,             # interpolated; no single quantile array
    probs      = NULL,
    scenario_id = sid,             # principal scenario for caller
    F_key       = F_key_near,
    k_nearest   = k_near,
    N_nearest   = n_near,
    q_true_brackets = c(q_lo, q_hi),   # NEW: report the bracket
    q_true_weight   = w,                # NEW: report the interp weight
    M1_nearest  = m1_near,
    clamp_notes = clamp_notes
  )
}
```

The caller `position_on_surface()` currently uses the returned
`quantiles` for band probabilities; that path needs a parallel
adjustment (interpolate the quantile array element-wise across
the two cohorts, or take the principal cohort's quantiles for
band-probability purposes and document that the `percentile`
field is interpolated while `band_probabilities` is anchored to
the snapped cohort).

## Risk surface

- **Existing regression tests will need bracket adjustment.** The
  v0.5.2 anchor tests use brackets like "20-80" or "<20"; the
  interpolated values may shift by 1-3 pp at panels near grid
  points. The Soares anchor (~9 pct) is in the floor region and
  shouldn't move; the §4.3 anchor (~50 pct) is at q = 0.97
  exactly and shouldn't move. The k = 25 row 6 panel
  (q_hat = 0.94) is the obvious test of the new behavior — should
  go from 0 (current snap) to a centered value in the 30-70 range
  (the interpolated reading).
- **Band probabilities require a separate decision.** The cleanest
  path is to interpolate the entire quantile array between the
  two cohorts and re-derive band probabilities from the
  interpolated array; the slightly-cheaper alternative is to
  return the principal cohort's quantiles unchanged and have only
  the `percentile` field be interpolated. Decide before
  implementation.
- **Vignette + paper App D need an updated description of the
  lookup logic.** The "snap to nearest q_true" sentence in the
  paper would change to "interpolate between the two adjacent
  q_true cohorts."

## Decision pending

- **Ship target.** v0.5.3 once v0.5.2 is out and absorbed.
- **Backward-compat consideration.** The interpolation changes
  the numerical percentile output for many cells. This is a
  documented behavioral change — opt-out flag or minor-version
  bump? Recommendation: minor-version bump (v0.6.0) to signal the
  break, with v0.5.3 as the previous-stable line for reproducing
  the published paper percentiles exactly.
- **Alternative.** Keep current snap behavior, add a global
  option `grass_options("lookup_method" = "interpolated")` that
  callers opt into. Less clean conceptually but lower migration
  risk.
