# grassr v0.7.0 calibration program (ratified 2026-07-04)

Goal: eliminate design snapping as a structural feature, kill the known
calibration biases, and stage the realism/scope programs so each tier
waterfalls into the next. Ratified by Austin 2026-07-04 ("goal right now
is 0.7.0 tier 1"). Compute: laptop (~12 cores, R 4.3.1) + Sebastian
(Mac Mini, 10 cores, R 4.5.2) over Tailscale.

Staging lives at `grassr/simulation/v070_program/{tier1,tier2,tier3}`.
Each tier's stages run in order; the runner advances on DONE markers.
Tier 2 and Tier 3 are spec'd but GATED: their scripts are written only
after the prior tier's results are reviewed, and the runner requires an
`APPROVED` file (Austin's explicit go) before entering a gated tier.

## Tier 1 — grid completeness (release target: 0.7.0)

Ordering constraint (load-bearing): thresholds are calibrated THROUGH
surface-percentile lookups, so the surface must densify first or the
threshold table is stale on arrival.

- **Stage 1 — surface N-densification + q-unification.**
  New N ∈ {15, 20, 30, 75, 100, 150, 300, 500} added to the existing
  {50, 200, 1000}; q-grid unified across k to the canonical set plus
  {0.92, 0.94, 0.97} (today only k=2/k=25 carry the high-q augments).
  Grid built by anti-join against the shipped
  `empirical_q_hat_surface$index`, so existing cells are never
  re-simulated and new cells inherit the exact (F_key, q, k) structure.
  ~34k new scenarios x 2,000 reps ~ 68M panels. Oracle ICC (glmer
  disabled), same metric set as v0.5.2 augmentations.
- **Stage 2 — surface rebuild + regression anchors.** Rebuild
  `empirical_q_hat_surface` from all per-scenario sources (original +
  v0.5.2 augments + Stage 1). Existing cells must reproduce bit-exactly
  at 5-decimal storage; the 28 sysdata regression anchors must pass
  unchanged. Produces a CANDIDATE sysdata artifact — nothing enters the
  package until Stage 5.
- **Stage 3 — delta-hat threshold grid, q-conditioned.**
  k ∈ {2,3,5,6,8,10,15,25} x N ∈ {15,...,1000} x q ∈ {0.65, 0.75,
  0.85, 0.92, 0.97}, 5 prevalences x 6 asymmetry levels x 1,000 reps
  per cell, computed against the Stage-2 surface. The q axis is the
  bias check: the shipped table was calibrated at q = 0.85 only. If
  thresholds are flat in q we have proven the current table unbiased;
  if not, the lookup becomes t(k, N, q_hat). Runs only after Stage 2.
- **Stage 4 — threshold table + LOO snapping analysis.** Rebuild the
  lookup table (size-alpha cuts at 5%/1%), rerun the leave-one-out
  interpolation-vs-snap comparison on the dense grid, decide
  interpolate-vs-snap on evidence. Fix lookup semantics: retire the
  both-finite rule (calibrated caution-only cells are used; missing
  divergent reported as "not calibrated" rather than borrowed), print
  snap distance + hull-exit warning on the card, fix the stale grid
  comment in delta_thresholds.R.
- **Stage 5 — assembly + verification.** Candidate sysdata assembled;
  full test suite; paper-coupled numbers re-verified (card table, §4
  examples, Klein, Hinde); size projection vs the CRAN caps (below).
  Output: 0.7.0 release candidate. CRAN timing: submit no sooner than
  ~1 month after 0.6.2's acceptance (rapid-fire updates are frowned on).

### sysdata size (the lurking constraint)
Cells grow ~10,140 -> ~44k (~4.3x). Shipped 5-decimal rda is 2.83 MB;
naive projection ~8-12 MB — over the 5 MB soft cap and near the 10 MB
tarball ceiling. Mitigations, in preference order, decided at Stage 5
with measurements: (1) integer quantization of the quantile arrays
(measured 35% saving at 0.6.1-scale; requires a decode layer — the
cost we declined at 0.6.2 becomes worth paying at 4x data); (2) drop
the krippendorff_a surface slice (20% of the quantile array; keep the
closed-form path); (3) coarser prob grid only as a last resort; (4) a
separate data package only if (1)-(3) fail.

## Tier 2 — DGP realism (gated; release target: 0.8.0)
Sensitivity discipline: measure misspecification cost, publish the
bound, do not silently expand the estimand.
- Item-difficulty arm (rater error varying with latent item difficulty)
  -> percentile-drift bounds.
- Correlated-error arm (pairwise error correlation beyond latent class).
- TPR breadth: single-bad-rater vs half-panel patterns, across q.
- Bootstrap-CI coverage + MC-SE audit at every published cell.
- One end-to-end sysdata build pipeline script (retire the
  augmentation-script sprawl).

## Tier 3 — scope growth (gated; 0.9/1.0 line)
- Dedicated intra-rater calibration cube (occasions t ∈ {2,3,5}, drift,
  within-rater correlation).
- Incomplete/unbalanced designs (mean raters per subject + overlap
  structure) — the largest adoption unlock, month-scale sims.
- grass_design() planning tool (invert the calibration).
- Gold-standard-subset API (turns Dawid-Skene into a shared-bias test
  when external truth exists).
- F-key compendium (skew-normal, t, bimodal) — future-paper material.

- 2026-07-04 (Austin, SUPERSEDES the band/envelope direction): **delta-hat

  the panel's delta-hat against its percentile on the matched null — that
  makes too much sense"). Stage-4 audit showed the null is a mixture at
  small N (stable body + degenerate-panel tail contamination), making cut
  lines non-monotone and their extreme quantiles unstable (bootstrap CIs up
  to 10pp). The thresholds TABLE is retired as the primary object; sysdata
  ships the null ECDF per (k, N, q) cell (13-quantile compression, same as
  coefficient surfaces); flags become percentile conventions (>=95th
  caution, >=99th divergent) evaluated on the matched null. Envelope/band
  machinery dissolves into the framework's existing cell-matching
  convention. PREREQUISITE before implementation: verify the tail
  contamination is constant-rater panels and re-derive the null through the
  production grass_report() pipeline (which drops constant raters).
  Teaching artifact: delta_null_distributions.html (published 2026-07-04).
