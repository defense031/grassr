The grass ecosystem

One package, one entry point

grass is a single package with a single user-facing reporting call:

grass_report(data, spec = grass_spec_binary(), ...)

grass-ecosystem.R

The spec argument selects the metric family. Today only grass_spec_binary() returns a usable spec; the other constructors (grass_spec_ordinal(), grass_spec_multirater(), grass_spec_continuous()) are placeholders that produce valid spec objects but error with a pointer to ?grass_roadmap when passed to grass_report(). This is deliberate: users can read the planned API and write code that will be ready for the release, without the package pretending to compute numbers it cannot yet compute.

Why one package, many families

Inter-rater and test-retest reliability are not one problem. A binary diagnosis, an ordinal severity scale, a continuous assay, and the internal consistency of a survey each live in a different statistical literature with a different reference standard. The grass framework treats them as branches of a common problem — “what value should this agreement coefficient take given the study’s actual parameters?” — and attaches a prevalence-conditioned reference curve to each.

The alternative, a package per family, fragments the user’s workflow without improving the science. A one-package design with internal dispatch means:

  • Existing user code does not change when a new family lands.
  • The same verbs — grass_methods(), grass_format_report(), plot(), tidy() — apply across every family.
  • The user does not have to remember which package exports what.

The framework labels GRASS, TURF, MEADOW, FIELD exist in the paper and in ?grass_roadmap as organising concepts. They never appear as function prefixes the user has to learn.

reference_level is the Se = Sp operating point on the analytical diagonal. Four bands ship — 0.70, 0.80, 0.85, 0.90 — each defined in closed form under conditional independence of the two raters given the true class.

Manuscript-ready methods paragraph

cat(grass_methods(r))
#> Inter-rater agreement between two raters on a binary outcome (N = 200 rating pairs) was analysed under the GRASS framework (Guide for Rater Agreement under Structural Skew), with reporting aligned to the Guidelines for Reporting Reliability and Agreement Studies [Kottner et al. 2011](https://doi.org/10.1016/j.jclinepi.2010.03.002). We computed Cohen's κ, PABAK (prevalence- and bias-adjusted κ), and Gwet's AC1 with Wilson-logit 95% confidence intervals on κ. Disease prevalence was estimated from the two raters' marginal positive rates as 0.50. Observed agreement was: Cohen's κ = 0.76 (95% CI 0.58, 0.87), PABAK = 0.76, Gwet's AC1 = 0.76. The (PI, BI) regime was classified as *balanced* (PI = 0.000, BI = 0.020). Observed agreement was compared against the prevalence-conditioned reference curve for raters operating at Se = Sp = 0.85 (the analytical expected value of each coefficient under conditional independence of the two raters given the true class). Youden's J at this reference operating point is 0.70. Because Cohen's κ, PABAK, and Gwet's AC1 are known to disagree at skewed prevalences under Landis-Koch fixed cutoffs, we report the signed distance from this analytical reference rather than a single static interpretation band (Semmel 202X).

The paragraph is GRRAS-compliant (Guidelines for Reporting Reliability and Agreement Studies), fills in the study’s numbers, and is safe to paste into an .Rmd or LaTeX manuscript. When the ordinal, multirater, or continuous families ship, each implements its own grass_methods() method with the paragraph appropriate to that design; the user call does not change.

Tomorrow: the rest of the framework

?grass_roadmap lays out the planned families. The release cadence is one paper and one minor release per family. Every family shares the core commitment: the interpretation of an agreement coefficient must be conditioned on the study’s actual parameters (prevalence, design, scale), not applied from a fixed 1977 cutoff table.
