PABAK reports 0.89, κ reports nearly zero, AC1 lands near 0.94. The regime is prevalence-dominated. The raters have essentially never been tested on the minority class, so high PABAK does not imply substantial agreement — it implies there is nothing to disagree about. Report all three metrics together with PI, and the reader can form their own judgment.

Is the drift from N or from structure?

A natural question when you see a study plotted on a reference curve: the observed points land off the lines — is that because the sample is small, or because something structural is going on?

Short answer: small N widens the scatter; prevalence drives the drift. The two effects are separable, and you can see the separation directly. We compare the same data-generating process at small N vs large N, then the same small N at balanced vs skewed prevalence.

# Helper: simulate two raters at Se = Sp = q against a latent truth.
draw <- function(n, p, q, seed = 1) {
  set.seed(seed)
  truth <- rbinom(n, 1, p)
  r1 <- ifelse(truth == 1, rbinom(n, 1, q), rbinom(n, 1, 1 - q))
  r2 <- ifelse(truth == 1, rbinom(n, 1, q), rbinom(n, 1, 1 - q))
  data.frame(r1 = r1, r2 = r2)
}

Same skew, different N

Same prevalence (p = 0.10), same rater quality (Se = Sp = 0.85), different sample sizes. The observed points stay in roughly the same spot relative to the reference lines — kappa below, PABAK/AC1 above — and the spot does not chase zero as N grows. N tightens the scatter but does not explain the drift.

The deltas between observed and reference are driven by the structure of the problem at p = 0.10 (kappa getting squeezed by its chance term), not by the 50-vs-500 sample size.

Same N, different prevalence

Now hold N = 50 constant but move prevalence from 0.10 (skewed) to 0.50 (balanced). At balanced prevalence the three metrics collapse toward each other and toward their reference curves. The earlier “kappa low, PABAK/AC1 high” pattern vanishes — it was a prevalence effect, not a noise effect.

r_skew <- grass_report(draw(n = 50, p = 0.10, q = 0.85), reference = "high")
r_bal  <- grass_report(draw(n = 50, p = 0.50, q = 0.85), reference = "high")
plot(r_skew, labels = "legend") + ggplot2::labs(title = "Prevalence = 0.10 (skewed)")
plot(r_bal,  labels = "legend") + ggplot2::labs(title = "Prevalence = 0.50 (balanced)")

Takeaway

When the three metrics land in visibly different positions relative to their reference curves, that is the algebra of the prevalence regime — not a signal that you need more data. Increasing N will tighten each point toward its own reference, but it will not line them up with each other. Report all three with PI, BI, and the regime label, and the reader can tell which effect is which.

What to report

Always together:

  1. Cohen’s κ (with a CI that respects the [-1, 1] bound at small N)
  2. PABAK
  3. Gwet’s AC1
  4. Prevalence index (PI)
  5. Bias index (BI)
  6. The regime label

grass_format_report(result) produces a one-line version of the same information suitable for pasting into a manuscript sentence or a results table cell.
